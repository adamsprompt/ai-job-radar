# Contributing to Powerful Assistants

Thank you for your interest in contributing. We are focused on building production-grade, Powerful Assistants that solve real organizational friction. 

## Development Protocol
To ensure high-fidelity execution and maintain the integrity of our codebase, please adhere to the following workflow:

1. **Strategic Alignment:** Before opening a pull request for a major feature, please open an Issue to discuss the business ROI and technical approach. 
2. **Branch Naming:** Use functional prefixes: `feat/` for new capabilities, `fix/` for bug resolution, and `docs/` for architectural updates.
3. **Commit Messages:** Use clear, declarative statements (e.g., "Add Recharts dependency for dashboard visualization").
4. **Security Check:** Ensure no secrets, API keys, or `.env` files are included in your commits. 

By contributing, you agree that your code will be licensed under the repository's MIT License.
